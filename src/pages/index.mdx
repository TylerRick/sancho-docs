### Getting started

#### Installation

Install Sancho using Yarn or NPM.

```
yarn add sancho
```

Import Sancho components into your project and wrap your project with the `ThemeProvider`.

```.jsx
import { Button, ThemeProvider } from "sancho";

function Project() {
  return (
    <ThemeProvider>
      <Button>Hello world!</Button>
    </ThemeProvider>
  );
}
```

#### Styling and Emotion

Sancho uses CSS-in-JS and [Emotion](https://emotion.sh/docs/introduction) for styling, and itâ€™s recommended (though not required) to use Sancho within projects that use Emotion. Emotion allows us to style components using the `css` prop - a bit like styling elements using the `style` prop, but with the benefits of pseudo selectors and the full power of css. Emotion also provides powerful and predictable composition. This means that you can pass css attributes into our components to override their styles.

```.jsx
/** @jsx jsx */
import { jsx } from "@emotion/core";
import { Button } from "sancho";

function MyCustomButton() {
  return (
    <Button css={{ color: "red" }}>I have red text!</Button>
  );
}
```

Most Sancho components provide BEM-style class names. This allows you to configure the styles of elements contained within components.

```.jsx
/** @jsx jsx */
import { jsx } from "@emotion/core";
import { Alert, theme } from "sancho";

function MyCustomAlert() {
  return (
    <Alert
      title="Hello world!"
      css={{
        maxWidth: "100vw",
        "& .Alert__bar": {
          display: "none"
        },
        "& .Alert__content": {
          padding: theme.spaces.sm
        }
      }}
    />
  );
}
```

For a full explanation of why Emotion was chosen, read this [blog post](https://benmcmahen.com/why-i-choose-emotion/). You can learn more about using the `css` prop on the [Emotion](https://emotion.sh/docs/css-prop) website.

#### Theme

Sancho uses a main `theme` object which contains global theming variables related to typography, color, spacing, and breakpoints. To access the theme, use the provided `useTheme` hook. View the [default theme here](https://github.com/bmcmahen/sancho/blob/master/src/Theme/index.ts).

```.jsx
/** @jsx jsx */
import { jsx } from "@emotion/core";
import { useTheme } from "sancho";

function Component() {
  const theme = useTheme();
  return (
    <div
      css={{
        padding: theme.spaces.sm,
        color: theme.colors.text.muted,
        [theme.breakpoints.md]: {
          padding: theme.spaces.md
        }
      }}
    >
      Hello world
    </div>
  );
}
```

The default theme comes with a dark and light mode. You can use `DarkMode` and `LightMode` to toggle between the two.

```.jsx
/** @jsx jsx */
import { jsx } from "@emotion/core";
import { DarkMode, LightMode } from "sancho";

function Component() {
  return (
    <DarkMode>
      <Text>This text will be white</Text>
      <LightMode>
        <Text>But this text will be dark</Text>
      </LightMode>
    </DarkMode>
  );
}
```

You can customize the default theme by overriding the properties you want.
The example below creates custom colors by providing [palx](https://palx.jxnblk.com/) generated color scales to the `generateColorsFromScales` function.

```.jsx
import { generateColorsFromScales, ThemeProvider, defaultTheme } from "sancho";
import palx from "palx";

const scales = palx("#08e");
const colors = generateColorsFromScales(scales);

const theme = {
  ...defaultTheme,
  ...colors,
  fonts: {
    ...defaultTheme.fonts,
    sans: "Roboto, sans-serif",
    base: "Roboto, sans-serif"
  }
};

function App() {
  <ThemeProvider theme={theme}>
    <MyAppContents />
  </ThemeProvider>
}
```
